================================================================================
PANEL & EVALUATION FLOW
================================================================================

The Panel module manages evaluation panel members and their evaluation workflow.
Panel members are invited to the tenant, assigned to programs (optionally to
specific batches), and then evaluate applications using configured scoring
criteria.

FLOW DIAGRAM:
--------------------------------------------------------------------------------

+----------------------------------------------------------------------+
|                    EVALUATION WORKFLOW                                 |
+----------------------------------------------------------------------+
|                                                                       |
|  1. Admin invites panel member (creates user if needed)               |
|  2. Admin assigns panel member to program (optional: specific batch)  |
|  3. Admin moves batch to UNDER_EVALUATION                             |
|  4. Panel member logs in, sees pending evaluations                    |
|  5. Panel member opens evaluation form for each application           |
|  6. Panel member scores each question and submits                     |
|  7. First evaluation: app status -> EVALUATED, score = total          |
|  8. Subsequent evaluations: score = average of all evaluations        |
|  9. Admin reviews evaluation summary and decides (verify/reject)      |
|                                                                       |
+----------------------------------------------------------------------+

SCORING CALCULATION:
--------------------------------------------------------------------------------

  Each evaluation question has:
    - weightage (float, e.g. 0.25 = 25% weight)
    - options with scores (e.g. 1-5)

  Per-answer score = option.score * question.weightage
  Total evaluation score = SUM of all per-answer scores

  After first evaluation:
    application.score = evaluator's totalScore
    application.status = EVALUATED

  After subsequent evaluations:
    application.score = AVERAGE of all evaluators' totalScores

================================================================================
API ENDPOINTS - ADMIN (Incubation Portal)
================================================================================

1. INVITE PANEL MEMBER
================================================================================

POST /api/incubation-portal/panel/invite

Description: Create or invite a panel member. If user doesn't exist,
             creates both ecosystem user and incubation user.

Headers: tenantkey: {tenant-key}
Auth: Required (Portal Token)

Body:
{
  "name": "Dr. Sharma",
  "email": "dr.sharma@evaluator.com",
  "password": "securepassword123"
}

Response:
{
  "success": true,
  "data": {
    "id": "cuid",
    "name": "Dr. Sharma",
    "email": "dr.sharma@evaluator.com",
    "isPanelMember": true
  }
}

Notes:
  - If user already exists as incubation user, marks isPanelMember=true
  - If user exists in another tenant, creates tenant membership
  - Creates ecosystem user with INCUBATION_PERSON role if needed

================================================================================

2. GET ALL PANEL MEMBERS
================================================================================

GET /api/incubation-portal/panel/members?tenantKey={key}

Headers: tenantkey: {tenant-key}
Auth: Required (Portal Token)

Response: Array of panel members with their assigned programs

================================================================================

3. ASSIGN PANEL MEMBERS TO PROGRAM
================================================================================

POST /api/incubation-portal/panel/program/:programId/assign

Description: Assign one or more panel members to a program, optionally
             to a specific batch within the program.

Headers: tenantkey: {tenant-key}
Auth: Required (Portal Token)

Body:
{
  "userIds": ["panel_user_cuid_1", "panel_user_cuid_2"],
  "batchId": "cuid_optional"
}

Response:
{
  "success": true,
  "data": {
    "assigned": ["panel_user_cuid_1", "panel_user_cuid_2"],
    "errors": []
  }
}

Notes:
  - batchId is optional. If provided, panel member is assigned to that
    specific batch and can only evaluate applications in that batch.
  - If batchId is null, panel member can evaluate all applications in
    the program regardless of batch.
  - Unique constraint: [programId, panelMemberId, batchId]
  - Also marks the user as isPanelMember in tenant membership

================================================================================

4. REMOVE PANEL MEMBER FROM PROGRAM
================================================================================

DELETE /api/incubation-portal/panel/program/:programId/member/:panelMemberId

Description: Deactivate a panel member's assignment to a program

Auth: Required (Portal Token)

6. GET PENDING EVALUATIONS
================================================================================

GET /api/incubation-portal/panel/pending-evaluations?programId=&batchId=

Description: Get all applications pending evaluation by the logged-in panel
             member. Shows both pending and already-evaluated applications.

Auth: Required (Portal Token - panel member login)

Query:
  - programId (optional): Filter by specific program
  - batchId (optional): Filter by specific batch

Response:
{
  "success": true,
  "data": {
    "pending": [
      {
        "applicationId": "cuid",
        "startup": { "id": "cuid", "name": "TechCorp", "logoUrl": "..." },
        "program": { "id": "cuid", "title": "Accelerator 2026" },
        "batch": { "id": "cuid", "batchName": "Cohort 3", "batchCode": "C3", "status": "UNDER_EVALUATION" },
        "status": "UNDER_EVALUATION",
        "totalEvaluations": 0,
        "submittedAt": "2026-03-15T10:00:00Z",
        "updatedAt": "2026-04-10T14:00:00Z"
      }
    ],
    "evaluated": [
      {
        "applicationId": "cuid",
        "startup": {...},
        "program": {...},
        "batch": {...},
        "status": "EVALUATED",
        "totalEvaluations": 2
      }
    ],
    "summary": {
      "totalPending": 5,
      "totalEvaluated": 3,
      "programsAssigned": 2
    }
  }
}

Notes:
  - If panel member is assigned with a batchId, only sees applications
    from that batch
  - If assigned without batchId, sees all applications in the program
  - "pending" = not yet evaluated by this member
  - "evaluated" = already submitted evaluation

8. SUBMIT EVALUATION
================================================================================

POST /api/incubation-portal/panel/evaluate/:applicationId

Description: Submit evaluation scores for an application

Auth: Required (Portal Token - panel member)

Body:
{
  "answers": [
    {
      "questionId": "eq_cuid_1",
      "optionId": "opt4",
      "comment": "Strong innovation in AI-powered analytics"
    },
    {
      "questionId": "eq_cuid_2",
      "optionId": "opt3",
      "comment": "Market size is moderate but growing"
    }
  ],
  "remarks": "Overall strong application with room for improvement in market strategy"
}

Response:
{
  "success": true,
  "data": {
    "evaluationId": "cuid",
    "totalScore": 3.75,
    "applicationStatus": "EVALUATED"
  }
}

Validation:
  - Panel member must be assigned to the application's program (and batch if applicable)
  - Application must be in UNDER_EVALUATION or EVALUATED status
  - Each panel member can only evaluate once per application
  - All question IDs must be valid evaluation questions for the program

Score Calculation:
  answer_score = option.score * question.weightage
  total_score = SUM(all answer_scores)

Status Update:
  - First evaluation: status changes from UNDER_EVALUATION to EVALUATED
  - Subsequent evaluations: status stays EVALUATED, score becomes average

FRONTEND IMPLEMENTATION GUIDE
================================================================================

PANEL MEMBER DASHBOARD:
  1. Call GET /panel/pending-evaluations on dashboard load
  2. Show "pending" list with "Evaluate" button for each
  3. Show "evaluated" list as read-only reference
  4. Use summary stats for dashboard cards

EVALUATION FORM:
  1. Call GET /panel/evaluation-form/:applicationId
  2. Display startup info and application answers as context
  3. Render each evaluation question with radio/select options
  4. Allow optional comment per question
  5. Allow overall remarks
  6. Submit via POST /panel/evaluate/:applicationId

ADMIN EVALUATION REVIEW:
  1. From application detail, call GET /panel/evaluations/:applicationId
  2. Show each evaluator's scores in a table
  3. Call GET /panel/evaluation-summary/:applicationId for aggregated view
  4. Use breakdown for per-question analysis chart
