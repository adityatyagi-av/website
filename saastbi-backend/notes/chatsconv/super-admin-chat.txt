SUPER ADMIN CHAT
=================

Overview:
  Super admins have elevated access across the platform.
  They can view and manage conversations for support/moderation purposes.

Access Level:
  - Super admins can access any conversation for moderation
  - They do NOT appear as participants in conversations they moderate
  - Their personal inbox shows only their own USER conversations

Capabilities:
  - View any conversation's messages (for moderation)
  - Delete messages that violate platform rules
  - Access conversation metadata and participant info
  - Cannot impersonate entities or send messages as other users

Inbox Behavior:
  - Super admin personal inbox: standard USER scoping
  - Admin panel: can browse all conversations (separate admin endpoints)
  - No cross-contamination between admin view and personal inbox

Implementation Notes:
  - Admin endpoints are separate from user-facing chat endpoints
  - Admin access checked via user.role or platform-level permissions
  - Audit trail maintained for admin actions on conversations
  - Admin moderation does not trigger real-time notifications to participants

Security:
  - Admin actions are logged for accountability
  - Cannot modify message content (only delete)
  - Cannot create conversations on behalf of other users
  - Read access does not mark messages as read for participants

Future Considerations:
  - Broadcast messaging from platform to all users
  - System-level announcements within chat
  - Automated moderation triggers
  - Escalation workflows from user reports
