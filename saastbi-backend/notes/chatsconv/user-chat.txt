USER CHAT SYSTEM
=================

Overview:
  Personal user-to-user messaging.
  Users chat under their own identity (firstName, lastName, profilePhoto).

Entity Type: USER

Identity Resolution:
  - User -> user.firstName + user.lastName / user.profilePhoto
  - No entity abstraction; messages always show the user's own name

Privacy Controls:
  - Users can set messageSetting: EVERYONE | CONNECTIONS | NO_ONE
  - EVERYONE: anyone can message them
  - CONNECTIONS: only connected users can message (Connection table)
  - NO_ONE: nobody can initiate new conversations
  - Privacy only checked for USER->USER messaging, not entity messages

Blocking:
  - Users can block each other via Block table
  - Blocked users cannot send or receive messages
  - Checked bidirectionally (blocker/blocked)

Inbox Behavior:
  - Personal inbox shows ONLY conversations where participantType = "USER"
  - Entity conversations (STARTUP, PAGE, TENANT) never appear here
  - Scoped by participant1Type/participant2Type = "USER" AND participant1Id/participant2Id = userId

Who Can Send:
  - Any authenticated user (subject to privacy/blocking)
  - No role restrictions for personal messaging

Contextual Messaging:
  - USER->USER can have contextType: MENTORSHIP, SESSION, etc.
  - Context enables special chat initiation flows (startMentorChat, etc.)
  - Context type does not affect inbox visibility

Socket Rooms:
  - On connect: joins "user:{userId}" room
  - Personal notifications delivered to this room only

Permission Flow:
  1. User sends message with no entityId (defaults to USER type)
  2. canMessage checks:
     a. Neither party has blocked the other
     b. Receiver's messageSetting allows this sender
     c. If CONNECTIONS: checks Connection table for accepted connection
  3. If all pass, message sent as senderType=USER, senderId=userId
