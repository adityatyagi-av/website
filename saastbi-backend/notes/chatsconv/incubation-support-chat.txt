INCUBATION SUPPORT CHAT
========================

Overview:
  Support/mentorship conversations within the incubation context.
  Incubation users (startups in programs) can chat with mentors and
  program coordinators through designated channels.

Context Types:
  - MENTORSHIP: mentor-mentee relationship chats
  - SESSION: session-specific chats (time-bound)
  - INCUBATION: general incubation context

Mentor Chat Flow (startMentorChat):
  - Incubation user initiates chat with assigned mentor
  - Creates conversation with contextType=MENTORSHIP
  - Sender: INCUBATION_USER type (incubation user's id)
  - Receiver: USER type (mentor's userId)
  - Mentor sees this in personal inbox (as USER participant)
  - Incubation user sees this in incubation portal

Session Chat Flow (startSessionChat):
  - Chat linked to a specific mentoring session
  - Creates conversation with contextType=SESSION, contextId=sessionId
  - Participants resolved from session's mentorship record
  - Auto-resolves sender/receiver from session membership

Mentorship Chat Flow (startMentorshipChat):
  - Chat linked to an ongoing mentorship relationship
  - Creates conversation with contextType=MENTORSHIP, contextId=mentorshipId
  - Participants: incubation user + mentor from mentorship record

Visibility:
  - Mentor sees support chats in their personal USER inbox
  - Incubation user sees support chats in incubation portal
  - These are NOT visible in startup/page entity inboxes
  - Scoped by participantType matching

Permission:
  - Mentor chats bypass privacy settings (canMessage allows contextual)
  - No blocking applies to contextual/support chats
  - Both parties must be part of the mentorship/session relationship

Real-Time:
  - Notifications via "user:{userId}" for mentors
  - Notifications via "user:{incubationUserId}" for incubation users
  - Both rooms joined on socket connection
