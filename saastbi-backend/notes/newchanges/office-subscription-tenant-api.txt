================================================================================
OFFICE SUBSCRIPTION API — INCUBATION (TENANT) PORTAL
================================================================================

Base URL prefix:  /api/incubation-portal
Auth middleware:  authenticatePortal (requires Bearer access token + tenantkey
                  header that resolves to an ACTIVE Tenant)

Required headers on EVERY request:
  Authorization: Bearer <accessToken>
  tenantkey:     <tenantKey>           // resolves the active tenant
  Content-Type:  application/json      // for POST bodies

Common response envelope (all 2xx and error responses):
{
  "success": <boolean>,
  "status":  <httpStatusCode>,
  "data":    <object|array|null>,
  "message": <string|null>
}

Standard error responses:
  400 — { success:false, status:400, data:null, message:"<validation msg>" }
  401 — { success:false, status:401, data:null, message:"Unauthorized" / "HostName Not Found" }
  403 — { success:false, status:403, data:null, message:"Not authorized" }
  404 — { success:false, status:404, data:null, message:"Subscription not found" }

3. PAUSE SUBSCRIPTION
================================================================================
POST /api/incubation-portal/office/subscriptions/:bookingId/pause

Body: none

Side effects:
  - Calls Razorpay subscription pause.
  - Sets OfficeSubscription.status = "PAUSED" and pausedAt = now().

Success response (200):
{
  "success": true,
  "status":  200,
  "message": "Subscription paused",
  "data": { /* updated OfficeSubscription row */ }
}

Errors:
  400  Subscription not yet active
  403  Not authorized
  404  Subscription not found

================================================================================
4. RESUME SUBSCRIPTION
================================================================================
POST /api/incubation-portal/office/subscriptions/:bookingId/resume

Body: none

Side effects:
  - Calls Razorpay subscription resume.
  - Sets OfficeSubscription.status = "ACTIVE" and pausedAt = null.

Success response (200):
{
  "success": true,
  "status":  200,
  "message": "Subscription resumed",
  "data": { /* updated OfficeSubscription row */ }
}

================================================================================
5. CANCEL SUBSCRIPTION
================================================================================
POST /api/incubation-portal/office/subscriptions/:bookingId/cancel

Body (all optional):
{
  "cancelAtCycleEnd": <boolean>    // default true. When false, cancels immediately.
}

Side effects:
  - Calls Razorpay subscription cancel with cancel_at_cycle_end flag.
  - Sets OfficeSubscription.status = "CANCELLED", cancelledAt = now().
  - Note: Razorpay will emit subscription.cancelled webhook which also flips
          OfficeBooking.status to CANCELLED.

Success response (200):
{
  "success": true,
  "status":  200,
  "message": "Subscription cancelled",
  "data": { /* updated OfficeSubscription row */ }
}

================================================================================
NOTES FOR THE FRONTEND
================================================================================
- Subscriptions are created from the receiver (startup) side; the tenant portal
  only manages existing subscriptions.
- A subscription's lifecycle:
    PENDING       — created in Razorpay, awaiting first authorization.
    AUTHENTICATED — first auth payment captured.
    ACTIVE        — recurring charges underway.
    HALTED        — auto-charge failed; dunning starts.
    PAUSED        — explicitly paused (manual action).
    CANCELLED     — cancelled by tenant or auto-cancelled after 21 days HALTED.
    COMPLETED     — totalCount cycles paid, lifecycle complete.
    EXPIRED       — booking endDate passed without completion.
- `nextChargeAt` is the next attempted auto-debit timestamp.
- `remainingCount` decrements as Razorpay sends subscription.charged webhooks.
- After cancel, do NOT show "Pause/Resume" controls.
