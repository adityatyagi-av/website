================================================================================
ARCHITECTURE OVERVIEW
================================================================================

The Tenant subscription system handles:

1. PLAN BROWSING & SELECTION
   - View all available plans with "already purchased" indicator
   - Each plan shows upgrade/downgrade classification relative to current plan

2. SUBSCRIPTION LIFECYCLE
   - Create subscription -> Pay via Razorpay -> Verify payment -> ACTIVE
   - Automatic renewal via Razorpay recurring payments
   - Cancel subscription (at cycle end or immediately)
   - Change plan (upgrade/downgrade with proration)

3. BILLING DASHBOARD (Portal)
   - View current plan, next billing date, days remaining
   - View recent payment history
   - Make manual payments through the portal

4. RENEWAL REMINDERS
   - Email reminders sent automatically before renewal:
     - WEEKLY plans: 2 days before
     - MONTHLY plans: 2 days before
     - YEARLY plans: 20 days before

================================================================================
DATA MODELS
================================================================================

Plan:
  - id: string (cuid)
  - name: string (e.g., "Basic", "Pro", "Enterprise")
  - price: float (in INR)
  - type: "MONTHLY" | "YEARLY" | "WEEKLY"
  - maxUsers: number (default: 5)
  - maxStartups: number (default: 1)
  - storageLimit: number | null
  - features: string[] (list of feature descriptions)
  - razorpayPlanId: string | null (Razorpay plan reference)
  - planModules: PlanModule[] (included modules)

Subscription:
  - id: string (cuid)
  - tenantId: string
  - planId: string
  - razorpaySubscriptionId: string (unique)
  - razorpayPaymentId: string | null
  - razorpaySignature: string | null
  - status: "PENDING" | "ACTIVE" | "COMPLETED" | "CANCELED" | "EXPIRED"
  - startDate: DateTime | null
  - endDate: DateTime | null (also serves as next billing date)
  - createdAt: DateTime
  - updatedAt: DateTime

Invoice:
  - id: string (cuid)
  - tenantId: string
  - invoiceNumber: string (unique, format: "INV-YEAR-HEXCODE")
  - amount: float
  - issuedDate: DateTime
  - dueDate: DateTime | null
  - paidDate: DateTime | null
  - status: "UNPAID" | "PAID" | "OVERDUE"
  - pdfUrl: string | null

BillingHistory:
  - id: string (cuid)
  - tenantId: string
  - amount: float
  - description: string | null
  - billingDate: DateTime
  - paymentMethod: string | null ("Razorpay" or "SYSTEM_CREDIT")
  - status: "SUCCESS" | "FAILED" | "PENDING"
  - invoiceId: string | null
  - createdAt: DateTime

        "id": "cuid_plan_2",

    "razorpaySubscriptionId": "sub_rzp_abc123",

- isPaymentDue (boolean): When true, show a "Pay Now" button prominently

    - "Overdue invoice" — an invoice with OVERDUE status exists

    - "Subscription paused - manual payment required" — Razorpay subscription paused
    - "Billing cycle expired" — daysUntilNextBilling reached 0
- isRenewalApproaching (boolean): True when renewal is near but not yet due
    - WEEKLY plans: within 2 days
    - MONTHLY plans: within 7 days
    - YEARLY plans: within 20 days
- renewalReminderText (string | null): Human-readable text e.g. "Your plan renews in 3 days"

Frontend Logic:
- if isPaymentDue === true → Show alert banner + "Pay Now" button using pendingAmount
- if isPaymentDue === false && isRenewalApproaching === true → Show soft info banner with renewalReminderText
- if both false → No payment action needed

6. CANCEL SUBSCRIPTION
================================================================================

Method: POST
URL: {{BASE_URL}}/api/incubation/subscription/:subscriptionId/cancel
Auth: Bearer Token Required

Purpose: Cancel the subscription. Can cancel at end of current cycle or
immediately.

Path Parameters:
- subscriptionId (string, REQUIRED)

Request Body:
{
  "cancelAtCycleEnd": true
}

- cancelAtCycleEnd: true → Service continues until endDate, then expires
- cancelAtCycleEnd: false → Service stops immediately, tenant SUSPENDED

Response (200):
{
  "success": true,
  "status": 200,
  "data": {
    "id": "cuid_sub_1",
    "status": "CANCELED",
    "endDate": "2026-04-01T00:00:00Z",
    "plan": { ... },
    "tenant": { ... }
  },
  "message": "Subscription cancelled successfully"
}

Frontend Notes:
- Show confirmation dialog: "Cancel at end of cycle?" vs "Cancel immediately?"
- cancelAtCycleEnd: true — tenant keeps access until endDate
- cancelAtCycleEnd: false — tenant loses access immediately
- After cancellation, redirect to plans page

URL: {{BASE_URL}}/api/incubation/subscription/change-plan

    "shortUrl": "https://rzp.io/i/new123",
    "changeType": "upgrade",
    "previousPlan": "Pro",
    "newPlan": "Enterprise",
    "proratedCredit": 547.91
  },
  "message": "Plan upgrade initiated successfully"
}

Error (400 — same plan):
{
  "success": false,
  "status": 400,
  "data": null,
  "message": "Tenant is already on this plan"
}

Frontend Notes:
- changeType: "upgrade" or "downgrade" — show appropriate messaging
- proratedCredit: amount credited for unused days on old plan
- shortUrl: open for user to pay for new plan
- Old subscription is CANCELED immediately
- Tenant stays ACTIVE during transition (not SUSPENDED)
- New subscription is PENDING until payment completes

10. VERIFY MANUAL PAYMENT (Portal)
================================================================================

Method: POST
URL: {{BASE_URL}}/api/incubation-portal/billing/verify-payment
Auth: Bearer Token + tenantKey header

Purpose: Verify the manual payment after Razorpay checkout completes.
Records the payment in billing history, creates an invoice, and
updates the subscription cycle dates.

Request Body:
{

================================================================================
COMPLETE FRONTEND FLOWS
================================================================================

Flow 1: View Plans & Subscribe (First Time)
  1. GET /api/incubation/plans/:tenantId
  2. Display plans — each has isAlreadyPurchased, changeType
  3. User selects a plan (changeType: "new")
  4. POST /api/incubation/create-subscription { planId, tenantId }
  5. Open shortUrl or use Razorpay checkout with razorpaySubscriptionId
  6. After payment: POST /api/incubation/verify-payment
  7. On success: redirect to dashboard, tenant is ACTIVE

Flow 2: Upgrade/Downgrade Plan
  1. GET /api/incubation/plans/:tenantId
  2. User selects a different plan (changeType: "upgrade" or "downgrade")
  3. Show confirmation: "Upgrade to Enterprise? Credit: INR 547.91"
  4. POST /api/incubation/subscription/change-plan { tenantId, newPlanId }
  5. Open shortUrl for new plan payment
  6. POST /api/incubation/verify-payment with new subscription details
  7. On success: redirect to dashboard

Flow 3: View Billing Dashboard (Portal)
  1. GET /api/incubation-portal/billing
  2. Display: current plan, next billing date, days remaining
  3. Display: recent payments table
  4. Check billing.isPaymentDue:
     - If true → Show alert with paymentDueReason + "Pay Now" button → Flow 4
     - If false, check billing.isRenewalApproaching:
       - If true → Show soft banner with renewalReminderText
  5. "Pay Now" button (always available) → Flow 4

Flow 4: Manual Payment Through Portal
  1. POST /api/incubation-portal/billing/create-order
  2. Open Razorpay checkout with orderId and razorpayKeyId
  3. After payment callback:
     POST /api/incubation-portal/billing/verify-payment
  4. On success: refresh billing dashboard

Flow 5: Cancel Subscription
  1. User clicks "Cancel Subscription"
  2. Show options: "Cancel at end of cycle" or "Cancel immediately"
  3. POST /api/incubation/subscription/:id/cancel { cancelAtCycleEnd }
  4. On success: show updated status

Flow 6: Check Next Billing Date
  1. GET /api/incubation/subscription/billing-info/:tenantId
  2. Display billing.nextBillingDate and billing.daysUntilNextBilling
  3. Display billing.remainingCount for total cycles remaining

================================================================================
WEBHOOK EVENTS (Server-to-Server, Razorpay -> Backend)
================================================================================

Endpoint: POST /api/incubation/webhook/razorpay
Auth: None (verified via x-razorpay-signature header)

Events handled:
- subscription.activated  -> Subscription set to ACTIVE
- subscription.charged    -> Invoice created, billing cycle dates updated
- subscription.completed  -> Subscription set to COMPLETED
- subscription.cancelled  -> Subscription CANCELED (tenant suspended only if
                             no other ACTIVE/PENDING subscription exists)
- subscription.paused     -> Tenant SUSPENDED
- subscription.halted     -> Tenant SUSPENDED

Note: subscription.charged also handles idempotency — duplicate webhook
events for the same payment ID are ignored.
