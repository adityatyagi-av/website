================================================================================
SUPER ADMIN — SUBSCRIPTION, PLAN & BILLING API DOCUMENTATION
================================================================================

Version: 1.0.0
Last Updated: 2026-03-29
Base URL: /api/super-admin
Subscription URL: /api/incubation (shared with tenant, but super admin has
  access to all subscriptions and plans)

Authentication: SuperAdmin Bearer Token (authenticate middleware)

================================================================================
ARCHITECTURE OVERVIEW
================================================================================

The Super Admin manages the complete subscription and billing lifecycle:

1. MODULE MANAGEMENT — Create and manage feature modules
2. PLAN MANAGEMENT — Create plans, assign modules, set pricing
3. TENANT MANAGEMENT — View tenants, their subscriptions, billing history
4. INVOICE MANAGEMENT — View all invoices across tenants
5. ADDON MANAGEMENT — Create addons, handle addon service requests
6. SUBSCRIPTION OVERSIGHT — View all subscriptions, manage lifecycle

================================================================================
DATA MODELS
================================================================================

Module:
  - id: string (cuid)
  - moduleName: string
  - moduleKey: string (unique identifier)
  - moduleDescription: string | null
  - isActive: boolean (default: true)
  - permissions: Permission[]

Plan:
  - id: string (cuid)
  - name: string
  - price: float (INR)
  - type: "MONTHLY" | "YEARLY" | "WEEKLY"
  - maxUsers: number (default: 5)
  - maxStartups: number (default: 1)
  - storageLimit: number | null
  - features: string[]
  - razorpayPlanId: string | null
  - planModules: PlanModule[]

Subscription:
  - id: string (cuid)
  - tenantId: string
  - planId: string
  - razorpaySubscriptionId: string (unique)
  - status: "PENDING" | "ACTIVE" | "COMPLETED" | "CANCELED" | "EXPIRED"
  - startDate: DateTime | null
  - endDate: DateTime | null
  - createdAt: DateTime

Invoice:
  - id: string (cuid)
  - tenantId: string
  - invoiceNumber: string (unique, format: "INV-YEAR-HEXCODE")
  - amount: float
  - issuedDate: DateTime
  - paidDate: DateTime | null
  - status: "UNPAID" | "PAID" | "OVERDUE"

BillingHistory:
  - id: string (cuid)
  - tenantId: string
  - amount: float
  - description: string | null
  - paymentMethod: string | null ("Razorpay" | "SYSTEM_CREDIT")
  - status: "SUCCESS" | "FAILED" | "PENDING"
  - invoiceId: string | null

Addon:
  - id: string (cuid)
  - addonKey: string (unique)
  - name: string
  - description: string | null
  - type: string (service type)
  - price: float | null
  - unit: string | null
  - estimatedPrice: string | null
  - icon: string | null
  - isActive: boolean (default: true)

AddonServiceRequest:
  - id: string (cuid)
  - tenantId: string
  - addonId: string
  - status: "PENDING" | "APPROVED" | "REJECTED" | "IN_PROGRESS" | "COMPLETED"
  - adminNotes: string | null
  - reviewedBy: string | null

1. CREATE PLAN
================================================================================

Method: POST
URL: {{BASE_URL}}/api/super-admin/create-plan
Auth: SuperAdmin Token Required

Purpose: Create a new subscription plan with modules, pricing, and limits.
Also creates the plan on Razorpay for recurring billing.

Request Body:
{
  "name": "Pro",
  "price": 999,
  "type": "MONTHLY",
  "moduleIds": ["cuid_mod_1", "cuid_mod_2"],
  "maxUsers": 20,
  "maxStartups": 5,
  "storageLimit": 10240,
  "features": ["20 Users", "5 Startups", "Priority Support"]
}

Response (201):
{
  "success": true,
  "status": 201,
  "data": {
    "id": "cuid_plan_2",
    "name": "Pro",
    "price": 999,
    "type": "MONTHLY",
    "maxUsers": 20,
    "maxStartups": 5,
    "storageLimit": 10240,
    "features": ["20 Users", "5 Startups", "Priority Support"],
    "razorpayPlanId": "plan_rzp_abc123",
    "planModules": [
      {
        "module": { "id": "cuid_mod_1", "name": "Core" }
      },
      {
        "module": { "id": "cuid_mod_2", "name": "Analytics" }
      }
    ]
  },
  "message": "Plan created successfully"
}

Frontend Notes:
- type must be: "MONTHLY", "YEARLY", or "WEEKLY"
- moduleIds is an array of existing module IDs
- razorpayPlanId is auto-generated during creation
- price is in INR

URL: {{BASE_URL}}/api/super-admin/get-tenants

Purpose: List all tenants with their current subscription, plan, active
user count, and next billing date. This is the main dashboard view.

Query Parameters:
- page (number, default: 1)

- search (string, optional) — Search by organization name or tenantKey
- sortBy (string, default: "createdAt")

================================================================================
COMPLETE FRONTEND FLOWS
================================================================================

Flow 1: Create & Manage Plans
  1. GET /api/super-admin/get-modules → list available modules
  2. POST /api/super-admin/create-plan → create new plan with modules
  3. GET /api/super-admin/get-plans → view all plans
  4. PUT /api/super-admin/update-plan-modules → adjust modules
  5. PUT /api/super-admin/update-plan-details → adjust limits/features

Flow 2: Monitor Tenant Subscriptions
  1. GET /api/super-admin/get-tenants → dashboard with all tenants
  2. Click tenant → GET /api/super-admin/get-tenant/:tenantId
  3. View: subscription status, billing history, invoices, users
  4. GET /api/incubation/subscription/billing-info/:tenantId for
     detailed next billing date

Flow 3: View & Manage Invoices
  1. GET /api/super-admin/get-invoice → all invoices dashboard
  2. Filter by status (PAID/UNPAID) or search by invoice number
  3. GET /api/super-admin/get-tenant/:tenantId/invoices → tenant-specific

Flow 4: Manage Subscriptions
  1. GET /api/incubation/subscriptions → all subscriptions
  2. Filter by status (ACTIVE/CANCELED/PENDING)
  3. POST /api/incubation/subscription/:id/cancel → cancel if needed

Flow 5: Addon Management
  1. POST /api/super-admin/create-addon → create new addon
  2. GET /api/super-admin/get-addons → list addons
  3. GET /api/super-admin/addon-requests → view tenant requests
  4. PATCH /api/super-admin/addon-requests/update-status → approve/reject
